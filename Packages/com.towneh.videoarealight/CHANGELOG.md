## [1.3.0] - 2026-04-28

### Added
- **Basis VR / Cilbox prop support.** `VideoAreaLightSource` auto-marks itself `[Cilboxable]` when `com.cnlohr.cilbox` is detected, letting the broadcaster run on Basis props. The asmdef references Cilbox by GUID so the package still compiles without it; opt out via `VAL_DISABLE_CILBOX` under Player → Scripting Define Symbols.
- **Per-volume Visibility Encoding mode.** Each `VideoAreaLightProbeVolume` now chooses between `Scalar` (the existing 1 byte/voxel format, default) and `Quadrant` (4 bytes/voxel, stores per-screen-quadrant visibility for correct directional shadows on walls and accurate per-screen-UV occlusion in the floor's specular reflection). Mix encodings per volume — typical pattern is Scalar for the venue-wide coarse volume and Quadrant for fine volumes around problem geometry where directional accuracy matters. The Demo sample uses both: `VAL_Probe_Coarse` and `VAL_Probe_Corridor` Scalar; `VAL_Probe_Fine` and `VAL_Probe_BoothFloor` Quadrant.
- Floor-reflection bug fixes in `VAL_SpecularContribution`: the dark band along the locus where `alpha + angularSpread = 1` (caused by a hard `saturate` introducing a derivative discontinuity) is replaced with a smooth `tanh`-based asymptotic cap; the bright wrap-around colour bands extending past the reflection's footprint (caused by the cookie texture's default Repeat wrap mode) are addressed with a defensive `saturate` on the cookie UV in the shader plus `EnforceCookieTextureSettings` in the demo builder that force-sets the placeholder PNG's import wrap mode to Clamp.
- `Tools > VideoAreaLight > Clean Up Old Sample Imports` — finds previous-version sample folders under `Assets/Samples/Video Area Light/` and deletes them after confirmation. Useful after package upgrades because Unity Package Manager doesn't auto-clean stale sample imports.
- New `Demo` sample (replaces `Example Scene`). One scene demonstrating the package's full feature set — area-light reflections on glossy floors, the zone mask cutting contribution at the venue boundary, cascading probe volumes catching what the OBB can't, and the new mixed Scalar/Quadrant encoding pattern. Generated by `Tools > VideoAreaLight > Build Demo Scene` after import; the builder also auto-bakes its probe volumes so the scene is play-ready.

### Changed
- Sample lineup consolidated to a single `Demo` plus the existing `Poiyomi Integration` template.
- Demo scene: preview capsules (`Player_1`, `Player_2`) under `VideoAreaLight/Lit` for at-a-glance lit-surface preview; tuned default screen intensity.
- **Cookie LOD ramp now scales with the source texture's mipmap count.** Previously hard-coded as `roughness * 7.0`, which on a 1080p RT (mip count ~11) sampled mip 7 at full roughness — still ~15×8 pixels of recognizable image content. The broadcaster now pushes `_VAL_CookieMipCount` from `Texture.mipmapCount`, and the shader ramps `roughness * (mipCount - 1)` so full roughness lands on the smallest mip (single-pixel average colour). At moderate roughness (e.g. 0.5), the cookie sample is also visibly softer than before — moves the package's reflection blur closer to typical IBL prefilter behaviour. **Caveat:** if the user-supplied cookie texture has Mipmaps disabled, `mipmapCount` is 1 and the ramp collapses to mip 0, giving a mirror-sharp reflection at every roughness. Fix is to enable Mipmaps + Auto-generate on the source RenderTexture / Texture2D.
- **Specular contribution gated linearly by smoothness.** At smoothness=0 the user expects no specular, only the diffuse polygon irradiance. The Karis MRP+GGX combination, by itself, produces a broad lobe at full roughness — a soft rectangular footprint tracking the screen's reflection geometry — even after the cookie has averaged out. Multiplying specular by smoothness fades that footprint to zero as smoothness drops. Behaviour change for materials sitting at low smoothness; rough materials lose the recognizable screen-shaped highlight and read as fully Lambertian under VAL. Diffuse path is unaffected.
- **Poiyomi integration: diffuse contribution now flows through `poiLight.finalLighting`** (instead of `finalLightAdd`), and the function-call template is bound to **`FRAGMENT_BASE_LIGHTING_EARLY`** (instead of `FRAGMENT_BASE_LIGHTING`). Diffuse now picks up Poi_Shading's `finalColor = baseColor * finalLighting` snapshot, which means `_LightingCap` and any other pre-snapshot pipeline adjustments apply to the VAL contribution. Specular continues through `finalLightAdd` (consumed in a separate anchor, so order-independent). After upgrading, re-run **Tools → VideoAreaLight → Install Poiyomi Module** *and* **VideoAreaLight - Apply to Material's Shader** to pick up both the new template and the new anchor binding.

### Added (HLSL)
- `VAL_EvaluateAreaLightSeparate` — public entry point in `RectAreaLight.hlsl` that returns diffuse irradiance with no albedo tint plus already-F0-tinted specular. Use this when integrating into a shader that multiplies a downstream lighting accumulator by baseColor on your behalf. The original `VAL_EvaluateAreaLight` is preserved as a thin wrapper, so the bundled `VideoAreaLight/Lit` shader and the Shader Graph node are unchanged.

### Added (Poiyomi)
- **Use Reflections & Specular Settings** toggle (default ON) in the VAL section. When ON and the Reflections & Specular module is enabled, VAL reads `poiFragData.smoothness` / `.metallic` / `.specularMask` (which already include the section's slider, Packed Maps, and inverts) and re-applies the Mochie *global* metallic / smoothness / specular-strength masks at the call site (the section writes those only to `mochieBRDFData`, not to `poiFragData`). Specular output is multiplied by the resolved `specularMask` to mirror the gating LTCGI applies, so the **Specular Visibility** slider, the Specular Mask channel of the Packed Maps texture, and `_MochieSpecularStrengthGlobalMask` now affect the VAL screen reflection.
- **Smoothness** and **Metallic** sliders in the VAL section, shown when **Use Reflections & Specular Settings** is OFF or the Reflections & Specular section is disabled (`_MochieBRDF == 0`). Lets users put VAL on a material that has the Reflections & Specular module turned off entirely, without VAL falling back to the default `poiFragData` values (smoothness/metallic = 1) which would produce a sharp mirror.

### Removed
- `Example Scene` sample. After upgrading, run `Tools > VideoAreaLight > Clean Up Old Sample Imports` (or manually delete `Assets/Samples/Video Area Light/<old-version>/`) to remove its imported copy.

### Notes for upgraders
- Existing scenes with `VideoAreaLightProbeVolume` components are unaffected — the new `encoding` field defaults to `Scalar`, which preserves all prior behaviour and storage formats. Authors who want the directional accuracy can opt in per volume.
- Cookie textures (anything assigned to a broadcaster's `Video Texture` field) should have their import Wrap Mode set to `Clamp`, not the Unity default `Repeat`. With Repeat, bilinear sampling at UVs near the texture's edges pulls colours from the opposite side and bleeds visible bands past the reflection's footprint. The Demo builder force-sets this on its generated placeholder; user-supplied textures need it set manually in the Inspector.

## [1.2.0] - 2026-04-27

### Added
- **Zone Mask occlusion.** Reference a BoxCollider on the broadcaster to bound where the screen's reflections can land. Cheap analytic test in the shader; the recommended primary tool for keeping reflections out of adjacent rooms.
- **Probe Volume occlusion.** New `VideoAreaLightProbeVolume` component bakes a 3D visibility texture for intra-room geometry a Zone Mask can't represent. Up to 4 cascade per scene, contributing multiplicatively where their bounds cover a fragment.
- **Static prop shadows.** Any collider inside a Probe Volume's bounds casts a baked shadow on receiving surfaces — no per-object setup.
- Visibility baker — `Tools > VideoAreaLight > Bake Visibility` (all volumes) and a "Bake This Volume" button on each volume's inspector. Uses Unity's Jobs system (`RaycastCommand.ScheduleBatch`) for parallel raycasts.
- Custom inspectors for `VideoAreaLightSource` and `VideoAreaLightProbeVolume`. The broadcaster shows an empty-state prompt with a "Create Zone Mask" button and a scene-wide listing of probe volumes with an "Add Probe Volume" button.
- `GameObject > Video Area Light > Zone Mask` and `> Probe Volume` menu entries.
- Authoring prefabs: `Runtime/Prefabs/VAL_ZoneMask.prefab` and `Runtime/Prefabs/VAL_ProbeVolume.prefab`.
- `Documentation~/Occlusion.md` — usage guide for both occlusion tools, including sizing recipes and a memory/voxel reference.
- `Documentation~/FutureIdeas.md` — forward-looking ideas not yet shipped.

### Changed
- Inspector tooltips, headers, and help text refreshed across both components.

### Removed
- `Runtime/Prefab/VideoAreaLight.prefab` (broadcaster prefab). Add the `VideoAreaLightSource` component directly to your screen mesh GameObject; the new prefabs at `Runtime/Prefabs/` cover the optional Zone Mask and Probe Volume.

## [1.1.0] - 2026-04-26

### Added
- **Poiyomi Pro Modular Shader integration.** Existing Poiyomi materials can receive light from a VideoAreaLight screen without switching shader.
- Two-step setup via the Tools menu: install the module once, then apply it to chosen materials. Confirmation dialog before each shader regeneration; multi-select supported via the Project window. Uninstall through the same menu.
- Per-material enable toggle (default off), diffuse/specular multipliers, and cookie checkbox.
- Integration menu items auto-hide when Poiyomi isn't installed.
- `Documentation~/PoiyomiIntegration.md` full guide; `Samples~/PoiyomiIntegration/` template for inspection or forking.
- Package Manager `documentationUrl`, `changelogUrl`, `licensesUrl` metadata.

## [1.0.0] - 2026-04-25

### Added
- Initial release.
- `VideoAreaLightSource` broadcaster — reads the video render texture's average colour and drives the per-pixel area-light effect via global shader uniforms.
- `VideoAreaLight/Lit` drop-in shader mirroring URP/Lit's property block, so material conversion is painless.
- Core lighting maths in `RectAreaLight.hlsl` — Karis MRP specular with a roughness-widened GGX lobe and analytic polygon irradiance for diffuse. No precomputed lookup textures or third-party data.
- Custom material inspector adds a Use Video Cookie toggle on top of URP/Lit's standard fields.
- Shader Graph entry point (`VideoAreaLight_SG.hlsl`) as an alternative to the drop-in shader.
- Pre-configured `Prefab/VideoAreaLight.prefab` with sensible defaults.
- `ExampleScene` sample demonstrating the effect on glossy walls and a wet-club-style floor.
